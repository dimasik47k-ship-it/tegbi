# Руководство по отладке

## Проблема 1: Кнопка "Подробнее" не работает и модальное окно не открывается

## Решение:

### Шаг 1: Замените BotCard.js на фиксированную версию

1. Переименуйте файл:
   ```
   components/BotCard.js → components/BotCardBackup.js
   ```

2. Переименуйте фиксированный файл:
   ```
   components/BotCardFixed.js → components/BotCard.js
   ```

## Проблема 2: На сайте отображается только 2 бота вместо 3

### Диагностика:

1. Откройте консоль браузера (F12)
2. Проверьте сообщения:
   - "Рендеринг карточки бота:" должно появиться 3 раза
   - "Загружено ботов: 3"
   - "Список ботов:" с массивом из 3 элементов

3. Проверьте DOM:
   - Откройте инспектор элементов (F12)
   - Найдите элементы с id="bot-card-1", id="bot-card-2", id="bot-card-3"
   - Проверьте, есть ли у них стили, которые скрывают их (display: none, visibility: hidden, opacity: 0)

4. Проверьте визуально:
   - Все карточки должны иметь красную границу (border-2 border-red-500)
   - Если вы видите только 2 красные границы, значит третья карточка не рендерится

### Шаг 2: Перезагрузите страницу

После замены файлов перезагрузите страницу в браузере.

### Шаг 3: Проверьте консоль браузера

1. Откройте консоль разработчика (F12)
2. Нажмите на кнопку "Подробнее"
3. Проверьте сообщения в консоли:

Ожидаемые сообщения:
```
Кнопка "Подробнее" нажата, бот: {id: ..., name: ..., ...}
onDetailsClick функция: function
handleDetailsClick вызван, бот: {id: ..., name: ..., ...}
```

### Шаг 4: Анализ результатов

Если в консоли есть сообщения, значит:
- ✅ Кнопка работает
- ✅ Функция передаётся правильно
- ✅ Обработчик вызывается

Если модальное окно всё равно не открывается, проблема в BotModal.js.

Если в консоли нет сообщений, значит:
- ❌ Кнопка не работает
- ❌ Функция не передаётся
- ❌ Обработчик не вызывается

## Дополнительная информация

### Проверка z-index модального окна

Если модальное окно открывается, но не видно, проверьте:
1. z-index должен быть достаточно высоким (например, 50)
2. Модальное окно должно быть вынесено за пределы main тега
3. Проверьте, нет ли перекрывающих элементов

### Проверка AudioContext

Ошибки AudioContext не влияют на работу модального окна, это только предупреждения браузера о политике автозапуска звука.
