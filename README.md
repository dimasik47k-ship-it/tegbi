## 📋 О проекте

Tegbi Catalog — это профессиональный каталог проверенных Telegram-ботов. Мы собираем, структурируем и предоставляем информацию о ботах, чтобы пользователи могли быстро найти надёжный инструмент для своих задач.

### 🔐 Принципы проекта

| Принцип | Описание |
|---------|----------|
| ✅ **Проверка** | Каждый бот проходит модерацию перед добавлением |
| 🔒 **Приватность** | Не собираем персональные данные пользователей |
| 🆓 **Бесплатно** | Полный доступ без регистрации и оплаты |
| 🚫 **Без рекламы** | Никакой навязчивой рекламы и отслеживания |
| 🌙 **Доступность** | Поддержка светлой и тёмной темы |

## ✨ Возможности

### Для пользователей

- 🔍 **Умный поиск** — фильтрация ботов по названию, описанию и функционалу
- 🎨 **Адаптивный дизайн** — корректное отображение на ПК, планшетах и смартфонах
- 🌓 **Тёмная тема** — автоматическое переключение или ручной выбор
- ✅ **Верификация** — бейдж «Проверено» для прошедших модерацию ботов
- 📖 **Инструкции** — подробные гайды по использованию каждого бота
- 📋 **Копирование** — быстрое копирование юзернейма бота в один клик
- 🔊 **Интерактивность** — плавные анимации и звуковые эффекты (опционально)

### Для разработчиков ботов

- 📝 **Предложить бота** — форма для добавления бота в каталог (`/botcreate`)
- 📋 **Требования** | Чёткие критерии принятия ботов
- ⏱ **Сроки** | Проверка заявки в течение 1-3 рабочих дней
- 🔗 **Прямая связь** | Обратная связь через Telegram

### Для интеграторов

- 📄 **Структурированные данные** | Все боты в формате JSON (`data/bots.js`)
- 🧩 **Простая интеграция** | Минимальные зависимости, чистая архитектура
- 🚀 **Готов к деплою** | Оптимизирован для Vercel

### Требования

- Node.js 18 или выше
- npm или yarn

### Установка

```bash
# Клонировать репозиторий
git clone https://github.com/dimasik47k-ship-it/tegbi.git

npm start

## ➕ Добавление нового бота

1. Откройте файл `data/bots.js`
2. Добавьте новый объект в массив `bots`:

```javascript
{
  id: 11,                          // Уникальный ID (число)
  name: "Название бота",          // Отображаемое имя
  username: "my_awesome_bot",     // Username без @
  avatar: "https://...",          // Ссылка на аватар (опционально)
  welcomeMessage: "Краткое описание бота...",  // Приветственное сообщение
  guideMarkdown: `## Инструкция\n\n1. Запустите бота\n2. Введите /start`, // Гайд в Markdown
  verified: true,                 // Флаг верификации (опционально)
  verifiedDate: "2026-01-20"      // Дата проверки (опционально)
}
```

3. Сохраните файл — изменения отобразятся автоматически (в режиме разработки)

> 💡 **Совет:** Перед добавлением убедитесь, что бот соответствует [критериям модерации](#-критерии-модерации).

## 🔐 Критерии модерации

### ✅ Принимается в каталог

- Бот публичный и стабильно работает
- Есть понятное описание и функционал
- Не нарушает [правила Telegram](https://core.telegram.org/api/terms)
- Не содержит спам, мошенничество или вредоносный код
- Разработчик готов к обратной связи

### ❌ Отклоняется

- Требует доступ к личным данным без обоснования
- Дублирует существующие боты без улучшений
- Описание не соответствует реальному функционалу
- Нет ответа от разработчика в течение 7 дней

### ⏱ Сроки проверки

- Стандартная заявка: **1-3 рабочих дня**
- Срочная заявка (с пометкой «Срочно»): **до 24 часов**

### 📧 Апелляция

Если ваш бот был отклонён, вы можете написать в поддержку: [@seraviellex](https://t.me/seraviellex)

## 🌐 Деплой на Vercel

### Автоматический деплой (рекомендуется)

1. Пушните код в репозиторий GitHub/GitLab
2. Импортируйте проект в [Vercel](https://vercel.com)
3. Настройки определяются автоматически (`next.config.js`)
4. Нажмите **Deploy** — сайт будет доступен через 1-2 минуты

### Переменные окружения (опционально)

| Переменная | Описание | Пример |
|------------|----------|--------|
| `TELEGRAM_BOT_TOKEN` | Токен бота для отправки заявок | `123456789:ABCdef...` |
| `TELEGRAM_ADMIN_CHAT_ID` | Ваш Chat ID для получения заявок | `987654321` |

> ⚠️ **Важно:** Никогда не коммитьте `.env.local` в репозиторий. Добавьте его в `.gitignore`.
